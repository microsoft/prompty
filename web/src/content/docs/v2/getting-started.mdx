## Installation

```bash
# Core + Jinja2 renderer + OpenAI provider
uv pip install prompty[jinja2,openai]==2.0.0a1

# With Azure OpenAI support
uv pip install prompty[jinja2,azure]==2.0.0a1

# Everything
uv pip install prompty[all]==2.0.0a1
```

## Write a `.prompty` File

Create `greeting.prompty`:

```yaml
---
name: greeting
model:
  id: gpt-4o-mini
  provider: openai
  connection:
    kind: key
    apiKey: ${env:OPENAI_API_KEY}
inputSchema:
  properties:
    - name: name
      kind: string
      default: World
template:
  format:
    kind: jinja2
  parser:
    kind: prompty
---
system:
You are a friendly assistant.

user:
Say hello to {{name}}.
```

## Run It

```python
import prompty

result = prompty.run(
    "greeting.prompty",
    inputs={"name": "Jane"},
)
print(result)  # "Hello Jane! How can I help you today?"
```

## Step-by-Step Pipeline

For more control, use the pipeline stages individually:

```python
import prompty

# Load the .prompty file → PromptAgent
agent = prompty.load("greeting.prompty")

# Render template + parse → list[Message]
messages = prompty.prepare(agent, inputs={"name": "Jane"})

# Call the LLM → raw response
response = prompty.execute(agent, messages)

# Extract clean result → string
result = prompty.process(agent, response)
```

## Async Support

Every function has an async variant:

```python
result = await prompty.run_async(
    "greeting.prompty",
    inputs={"name": "Jane"},
)
```

## Environment Variables

Use `${env:VAR_NAME}` in your `.prompty` files to reference
environment variables. Create a `.env` file in your project root:

```bash
OPENAI_API_KEY=sk-your-key-here
```

Prompty automatically loads `.env` files via `python-dotenv`.
