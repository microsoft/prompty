## Pipeline Functions

### `prompty.load(path) → PromptAgent`

Parse a `.prompty` file into a typed `PromptAgent` object.

```python
agent = prompty.load("chat.prompty")
print(agent.name)        # "chat"
print(agent.model.id)    # "gpt-4o"
```

### `prompty.render(agent, inputs) → str`

Render the template with inputs. Returns the raw rendered string
before parsing into messages.

```python
rendered = prompty.render(agent, inputs={"q": "Hi"})
# "system:\nYou are helpful.\n\nuser:\nHi"
```

### `prompty.parse(agent, rendered) → list[Message]`

Parse a rendered string into abstract messages.

```python
messages = prompty.parse(agent, rendered)
# [Message(role="system", ...), Message(role="user", ...)]
```

### `prompty.prepare(agent, inputs) → list[Message]`

Composite: render + parse + thread expansion. Returns
wire-ready messages.

```python
messages = prompty.prepare(agent, inputs={"q": "Hi"})
# [Message(role="system", ...), Message(role="user", ...)]
```

### `prompty.run(agent, messages) → result`

Composite: call the LLM executor then process the response.

```python
result = prompty.run(agent, messages)
# "Hello! How can I help you?"

# Pass raw=True to get the raw SDK response
response = prompty.run(agent, messages, raw=True)
# ChatCompletion(...)
```

### `prompty.process(agent, response) → result`

Extract clean content from a raw LLM response.

```python
result = prompty.process(agent, response)
# "Hello! How can I help you?"
```

### `prompty.execute(path_or_agent, inputs) → result`

Top-level orchestrator: load → prepare → run (all-in-one).

```python
result = prompty.execute("chat.prompty", inputs={"q": "Hi"})
```

### `prompty.validate_inputs(agent, inputs)`

Check that all required inputs (those without defaults) are
provided. Raises `ValueError` if any are missing.

### `prompty.headless(api, content, model, ...)`

Create a `PromptAgent` programmatically without a file:

```python
agent = prompty.headless(
    api="embedding",
    content="Hello world",
    model="text-embedding-ada-002",
    provider="openai",
    connection={"kind": "key", "apiKey": "sk-..."},
)
result = prompty.run(agent, agent.metadata["content"])
```

## Agent Mode

### `prompty.execute_agent(path_or_agent, inputs, tools, ...)`

Run a prompt with an automatic tool-calling loop.

```python
def get_weather(location: str) -> str:
    return f"72°F and sunny in {location}"

result = prompty.execute_agent(
    "agent.prompty",
    inputs={"question": "Weather in Seattle?"},
    tools={"get_weather": get_weather},
    max_iterations=10,  # default
)
```

**Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `prompt` | `str \| PromptAgent` | Path or agent |
| `inputs` | `dict` | Template inputs |
| `tools` | `dict[str, Callable]` | Tool functions |
| `max_iterations` | `int` | Loop limit (default 10) |
| `raw` | `bool` | Skip processing |

**Error handling:**

- Bad JSON in tool args → error sent to model
- Tool exception → error string sent to model
- Missing tool → error message, no crash
- Max iterations exceeded → `ValueError`

### `prompty.execute_agent_async(...)`

Async variant. Async tool functions are properly awaited.

```python
async def get_weather(location: str) -> str:
    return f"72°F in {location}"

result = await prompty.execute_agent_async(
    "agent.prompty",
    inputs={"question": "Weather?"},
    tools={"get_weather": get_weather},
)
```

## Connection Registry

### `prompty.register_connection(name, client=...)`

Register a pre-configured SDK client by name:

```python
from openai import AzureOpenAI
from azure.identity import (
    DefaultAzureCredential,
    get_bearer_token_provider,
)

client = AzureOpenAI(
    azure_endpoint=os.environ["AZURE_ENDPOINT"],
    azure_ad_token_provider=get_bearer_token_provider(
        DefaultAzureCredential(),
        "https://cognitiveservices.azure.com/.default",
    ),
)
prompty.register_connection("my-azure", client=client)
```

Then in your `.prompty` file:

```yaml
model:
  id: gpt-4o
  provider: azure
  connection:
    kind: reference
    name: my-azure
```

### `prompty.get_connection(name) → client`

Look up a registered connection. Raises `ValueError` if not
found.

### `prompty.clear_connections()`

Remove all registered connections. Useful in tests.

## Streaming

Set `stream: true` in model options:

```python
agent = prompty.load("chat.prompty")
messages = prompty.prepare(agent, inputs={...})

# Enable streaming
agent.model.options.additionalProperties = {"stream": True}

response = prompty.run(agent, messages, raw=True)
for chunk in prompty.process(agent, response):
    print(chunk, end="", flush=True)
```

The streaming processor handles:

- Content deltas (yielded as strings)
- Tool call deltas (accumulated, yielded at end)
- Refusal detection (raises `ValueError`)
- Empty heartbeat chunks (skipped)

## Structured Output

Define `outputSchema` to get JSON-parsed results:

```yaml
outputSchema:
  properties:
    - name: city
      kind: string
    - name: temp
      kind: integer
```

The processor automatically JSON-parses the response when
`outputSchema` is present.

## Tracing

```python
from prompty import Tracer, PromptyTracer, trace

# Add a JSON file tracer
Tracer.add("json", PromptyTracer("./traces").tracer)

# All pipeline calls are traced automatically
result = prompty.execute("chat.prompty", inputs={...})

# Trace custom functions
@trace
def my_function():
    ...
```

OpenTelemetry integration:

```python
from prompty.tracing.otel import otel_tracer
Tracer.add("otel", otel_tracer())
```
