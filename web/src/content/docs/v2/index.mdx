Prompty v2 is a ground-up rewrite of the Python runtime built on
[AgentSchema](https://microsoft.github.io/AgentSchema/). It keeps
the same `.prompty` file format you know, with a modernized
pipeline architecture and new capabilities.

## What's New

### AgentSchema Type System

Every `.prompty` file loads into a typed `PromptAgent` object from
AgentSchema. This gives you validated models, connections, tools,
and schemas — not raw dicts.

### Pipeline Architecture

The runtime is a four-step pipeline with swappable
implementations:

```text
execute()   →  top-level: load + prepare + run
  prepare() →  render + parse  →  list[Message]
  run()     →  executor + process → clean result
```

Each step is a protocol — you can bring your own renderer,
parser, executor, or processor.

### Agent Mode

`execute_agent()` provides a built-in tool-calling loop:

```python
result = prompty.execute_agent(
    "my-agent.prompty",
    inputs={"question": "What's the weather?"},
    tools={"get_weather": get_weather_fn},
)
```

Features: automatic tool execution, bad JSON recovery, error
handling, max iteration guards, and async support.

### Connection Registry

Pre-configure SDK clients (Azure AAD, managed identity, etc.)
and reference them by name in `.prompty` files:

```python
prompty.register_connection("my-azure", client=azure_client)
```

```yaml
model:
  connection:
    kind: reference
    name: my-azure
```

### Streaming Hardening

Tool call delta accumulation, refusal detection, and empty
chunk handling — all built into the processor.

## Requirements

- Python ≥ 3.11

## Install

```bash
pip install prompty[jinja2,openai]==2.0.0a1
```

> **Note:** This is an alpha release — the API may change before the stable
> v2.0.0 release. Install a specific version to pin your dependency.
