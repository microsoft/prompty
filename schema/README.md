# Prompty Schema

This directory contains the TypeSpec model definitions and code generation tooling for the Prompty type system. The TypeSpec models are the **source of truth** — all runtime code (TypeScript, Python, C#, Go, Rust) is generated from them.

## Structure

```
schema/
├── model/           # TypeSpec models (SOURCE OF TRUTH)
│   ├── main.tsp     # Entry point
│   ├── agent.tsp    # Prompty type (the root)
│   ├── model.tsp    # Model, ModelOptions
│   ├── connection.tsp
│   ├── properties.tsp
│   ├── template.tsp
│   ├── tools.tsp
│   └── tools/       # Tool subtypes (MCP, OpenAPI)
├── emitter/         # Code generation tooling (TypeSpec emitter)
├── tspconfig.yaml   # Generation targets
└── package.json     # Dependencies
```

## Regenerating Runtime Code

When you modify any `.tsp` file in `model/`, regenerate the runtime code:

```bash
cd schema

# First time only: install dependencies
npm install

# Build the emitter (if emitter source changed)
npm run build:emitter

# Generate all runtimes
npm run generate

# Or do both in one step
npm run build
```

This generates code into:
- `runtime/typescript/packages/core/src/model/` — TypeScript
- `runtime/python/prompty/prompty/model/` — Python
- `runtime/csharp/Prompty.Core/Model/` — C#
- `runtime/go/prompty/model/` — Go
- `runtime/rust/prompty/src/model/` — Rust
- `docs/model/` — Markdown documentation

## Generated Code

Generated files are **committed to the repo**. The emitter is a dev-time tool — consumers don't need TypeSpec installed.

Generated files have a header:
```
// WARNING: This is an auto-generated file. DO NOT EDIT THIS FILE DIRECTLY
```

## Adding or Modifying Types

1. Edit the relevant `.tsp` file in `model/`
2. Run `npm run build` from `schema/`
3. Verify runtime tests pass
4. Commit both the `.tsp` changes and generated output

## Origin

This emitter was originally part of [AgentSchema](https://github.com/microsoft/AgentSchema). The TypeSpec models have been trimmed to the Prompty subset (no Workflow, ContainerAgent, or AgentManifest types).
